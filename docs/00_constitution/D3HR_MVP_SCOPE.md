# D3HR MVP Scope

## Purpose

Lock MVP scope so development does not expand into ERP replacement, compensation automation, full community, or tenant-specific consulting software.

## MVP Definition

The MVP proves that one public institution can convert job data and daily work records into usable HR evidence.

## Included

### 1. NCS-to-JobDB

- NCS source mapping
- Institution job creation
- Task and competency mapping
- NCS-derived vs institution-modified distinction

### 2. Job Description Wiki

- Draft, review, approve, finalize, publish lifecycle
- Versioned job descriptions
- Reviewer comments
- Audit trail

### 3. Identity, Access, Workflow, Audit

- Tenant isolation
- Organization hierarchy
- MVP roles
- RBAC + ABAC + workflow-state policy foundation
- Audit logs for sensitive actions

### 4. Daily 3-Line Check-In

- Employee daily work note
- Goal/task linkage
- Evidence metadata
- Blocker/support request

### 5. OKR and Performance Evidence

- OKR cycle
- Objective and key-result linkage
- Evidence accumulation
- Manager feedback

### 6. Career Progress Foundation

- Career profile
- Job-fit placeholder
- Competency gap placeholder
- Career progress bar

### 7. HR Evidence Dashboard

- JobDB completeness
- Check-in/evidence completeness
- Approval backlog
- Evidence Pack readiness

### 8. Basic Admin

- Tenant management
- User/org import
- Role assignment
- Workflow configuration
- Audit-log viewer
- Data quality dashboard

## Excluded Until Later Phase

- Full ERP replacement
- Payroll or full compensation engine
- AI final evaluation score generation
- Automatic poor-performer labeling
- Full external community
- Full inter-institution benchmark marketplace
- Full workforce-planning simulator
- Institution-specific code forks

## MVP Success Criteria

The MVP succeeds when:

- HR can create and govern a versioned job database.
- Employees can record daily work evidence in under one minute.
- Managers can review scoped evidence and provide feedback.
- HR can see job-centered evidence readiness.
- Sensitive access is policy-gated and logged.
- Documentation and code remain synchronized.
