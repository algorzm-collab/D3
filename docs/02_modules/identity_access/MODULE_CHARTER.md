# Module Charter: Identity & Access

## Purpose

Control who can access, change, approve, export, and administer D3HR data.

## Owns

- Users
- Tenants
- Organizations
- Roles
- Role assignments
- Access policies
- Delegations
- Workflow permissions
- Audit logs

## Does Not Own

- Job taxonomy semantics
- Performance evaluation formulas
- Community moderation content policy

## MVP Scope

- Employee, Manager, HR, Institution Head, Consultant, System Operator roles
- Tenant and organization scoping
- RBAC + ABAC + workflow-state policy foundation
- Audit log foundation
