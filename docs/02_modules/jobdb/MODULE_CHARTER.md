# Module Charter: JobDB

## Purpose

Turn NCS and institution-specific job reality into versioned, operational job data.

## Owns

- NCS source mapping
- Job families
- Jobs
- Job versions
- Job-description element architecture
- Job-description field catalog
- Tasks
- Competencies
- Knowledge, skills, attitudes
- Job descriptions
- Job-description workflow and versioning

## MVP Scope

- NCS-to-JobDB mapping model
- Job-description wiki structure
- Structured job-description element groups
- Field catalog for strategic input, job core, requirement, job value, performance, competency, mobility, and talent type
- Versioned approval workflow
- Task and competency linkage
