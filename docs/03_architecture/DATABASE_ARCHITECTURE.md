# Database Architecture

## Principles

- Tenant isolation is mandatory.
- Job data is versioned.
- Sensitive HR data is auditable.
- Policy data is configurable, not hardcoded.
- Recommendations store evidence, not only scores.
- Community data is separated from internal HR data.

## Initial Schema Domains

- Identity and access
- JobDB
- Work and OKR
- Performance evidence
- Career and learning
- Workforce and placement
- Evidence packs
- Community and benchmark
- Admin operations
