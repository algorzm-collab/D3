# Documentation SOP

## Rule

Documentation is not optional. It is part of configuration management.

## Required Updates

- Feature change: module charter or module spec when scope changes
- Policy change: docs/04_policies and module POLICY
- DB change: DATA_MODEL
- API change: API_SPEC
- UI change: UI_SPEC
- Architectural decision: ADR
- Release: release note

## Recovery Use

Docs must be sufficient for a new CTO, developer, or AI agent to rebuild intent after context loss.
