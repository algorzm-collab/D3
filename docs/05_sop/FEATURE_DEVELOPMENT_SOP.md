# Feature Development SOP

## Steps

1. Create or update GitHub Issue.
2. Assign module.
3. Check product constitution.
4. Check do-not-build list.
5. Document policy and permission impact.
6. Document data model impact.
7. Document audit-log impact.
8. Add or update API/UI spec.
9. Implement.
10. Add tests or acceptance checks.
11. Update module docs.
12. Open PR with documentation links.
13. Pass code review and doc review.
14. Merge.
15. Update release note if user-facing.

## Definition of Done

Implementation without updated documentation is not done.
