# UI Conventions

## Role-Based Home

Each persona must have a different first screen.

- Employee: goals, check-in, career progress, feedback
- Manager: team progress, blockers, evidence, feedback queue
- HR: JobDB quality, approvals, evidence readiness, policy alerts
- Institution Head: HR risk briefing and aggregate indicators
- Consultant: project diagnostic board
- System Operator: tenant, usage, data quality, support state

## Screen Requirements

Every screen spec must define:

- Purpose
- Primary persona
- Data shown
- Role differences
- Empty state
- Loading state
- Error state
- Access denied state
- Audit implication, if any

## Product Language

Avoid labeling language for low performance.

Prefer:

- risk signal
- support needed
- job-fit issue
- competency gap
- workload imbalance
- evidence incomplete

Avoid:

- failure
- bad employee
- poor performer as a fixed identity
