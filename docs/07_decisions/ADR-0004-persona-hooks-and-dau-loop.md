# ADR-0004: Persona Hooks And DAU Loop

## Status

Accepted

## Decision

D3HR must preserve persona-specific hooks and daily/weekly usage loops as first-class product architecture.

The product must not become only an HR back-office database.

## Context

D3HR needs high MAU/DAU to accumulate real evidence data. Public-institution employees, managers, HR, institution heads, consultants, veterans, and evaluators all need different reasons to use the platform.

## Consequences

- Persona home screens are mandatory.
- Daily check-ins must return value to employees.
- Manager and HR screens must expose next actions, not passive dashboards.
- Community and benchmark features must connect back to job, career, or evidence data.
- Backend data linkage must support persona hooks.
